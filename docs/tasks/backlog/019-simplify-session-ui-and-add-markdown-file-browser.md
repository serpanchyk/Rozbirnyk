# 019 — Simplify Session UI And Add Markdown File Browser

## Goal

Refine the frontend session view so it focuses on useful simulation content
instead of internal identifiers, and make Markdown outputs easier to inspect in
place.

## Status

Backlog.

## Acceptance Criteria

- The UI no longer shows the session ID as a primary visible field in the main
  session workflow.
- The session view has one dedicated box/panel for Markdown outputs.
- That box lists the available Markdown files grouped or labeled clearly for
  timeline, state files, and actor files.
- Users can click a listed file and read its contents directly in the UI.
- The file viewer works for the current session without requiring users to know
  internal file paths.
- The layout remains usable on desktop and mobile.
