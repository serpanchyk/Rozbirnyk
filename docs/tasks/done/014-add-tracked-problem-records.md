# 014 — Add Tracked Problem Records

## Goal

Capture known operational issues in a dedicated `problems/` folder so they are
easy to hand off, prioritize, and resolve.

## Status

Done.

## Acceptance Criteria

- A `problems/` folder exists at the repo root.
- Each known problem has its own Markdown record.
- Problem records include current status, evidence, impact, and possible
  resolution paths.
